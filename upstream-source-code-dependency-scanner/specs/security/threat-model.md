# Threat Model — Upstream Source Code Dependency Scanner

**Version**: 1.0
**Last Updated**: 2026-03-12
**Status**: Complete

---

## 1. System Overview

The upstream dependency scanner is a C11 binary that reads RPM spec files, Go module files, Python project files, source tarballs, PRN reports, and CSV data files. It produces JSON manifests and patched spec files. The scanner runs in CI pipelines and on developer workstations.

**Trust boundary**: The scanner processes untrusted package metadata (spec files, go.mod, tarballs) that originate from upstream open-source projects and may be modified by package maintainers.

---

## 2. Attack Surface

| # | Input Type | Entry Point | Format | Trust Level |
|---|-----------|-------------|--------|-------------|
| AS-1 | RPM spec files | `spec_parse_directory()` | Text (RPM spec syntax) | Medium — maintained by Photon team, but sourced from upstream |
| AS-2 | Go module files (`go.mod`) | `gomod_parse_file()` | Text (Go module syntax) | Low — from upstream git repositories |
| AS-3 | Source tarballs (`.tar.gz`) | `tarball_extract_file()` | Binary archive | Low — from upstream, may contain malicious paths |
| AS-4 | PRN package report files | `prn_map_load()` | Text (structured report) | Medium — generated by Photon tooling |
| AS-5 | CSV data files | `gomod_map_load()`, `docker_sdk_map_load()`, `api_patterns_load()` | Text (CSV) | High — maintained in scanner repository |
| AS-6 | CLI arguments | `main()` getopt | String arguments | Low — user-supplied, may contain traversal |

---

## 3. Threat Actors

| Actor | Motivation | Capability | Likelihood |
|-------|-----------|------------|------------|
| **Malicious package maintainer** | Inject malicious dependencies into Photon packages via crafted spec files or go.mod | Can modify spec files, go.mod, upload tarballs; limited to specific packages | Medium |
| **Compromised upstream** | Supply chain attack via poisoned source tarballs or go.mod with path traversal | Full control of upstream repository; can inject symlinks, traversal paths, malicious archives | Medium |
| **Supply chain attacker** | Modify CSV mapping files or PRN reports to redirect dependency resolution | Access to build infrastructure or CI pipeline; can modify data files | Low |

## 4. STRIDE Analysis

### AS-1: RPM Spec Files

| Threat | Category | Risk | Mitigation | Status |
|--------|----------|------|------------ |--------|
| Malformed spec crashes parser | **D**enial of Service | Medium | Bounded buffers (`MAX_LINE_LEN`), `snprintf()` everywhere | Implemented |
| Macro expansion injection (`%{!?name:$(cmd)}`) | **T**ampering | Low | Macros are string-substituted, not shell-evaluated | Implemented |
| Oversized spec file exhausts memory | **D**enial of Service | Low | Fixed-size buffers, no unbounded allocation | Implemented |

### AS-2: Go Module Files (`go.mod`)

| Threat | Category | Risk | Mitigation | Status |
|--------|----------|------|------------ |--------|
| Malicious module path causes command injection | **E**levation of Privilege | High | `fork()/execlp()` with explicit args, no `system()` | Implemented |
| Oversized `go.mod` causes buffer overflow | **T**ampering | Medium | Bounded `fgets()` with `MAX_LINE_LEN`, `snprintf()` | Implemented |
| Crafted version string overflows constraint field | **T**ampering | Medium | `MAX_VERSION_LEN` (64 bytes), `snprintf()` | Implemented |

### AS-3: Source Tarballs

| Threat | Category | Risk | Mitigation | Status |
|--------|----------|------|------------ |--------|
| Path traversal in tarball entries (`../../etc/passwd`) | **T**ampering | Critical | `_is_safe_path_component()` rejects `..` and metacharacters | Implemented |
| Symlink escape in archive | **E**levation of Privilege | High | Extract to `mkstemp()` temp file, not to directory tree | Implemented |
| Zip bomb / tar bomb exhausts disk | **D**enial of Service | Medium | Extract single file only (`tar -O`), not full archive | Implemented |
| Shell injection via tarball filename | **E**levation of Privilege | High | `fork()/execlp()` with explicit arguments | Implemented |

### AS-4: PRN Package Report Files

| Threat | Category | Risk | Mitigation | Status |
|--------|----------|------|------------ |--------|
| Path traversal in repo/owner names | **T**ampering | Medium | `strstr(entry, "..")` rejection at `prn_parser.c:136` | Implemented |
| Malformed PRN entries cause incorrect mapping | **S**poofing | Low | Strict field parsing, skip malformed lines | Implemented |

### AS-5: CSV Data Files

| Threat | Category | Risk | Mitigation | Status |
|--------|----------|------|------------ |--------|
| Modified mapping redirects dependency resolution | **T**ampering | Medium | Data files are version-controlled in repository | Implemented |
| Malformed CSV causes parser crash | **D**enial of Service | Low | Bounded field parsing, skip malformed lines | Implemented |

### AS-6: CLI Arguments

| Threat | Category | Risk | Mitigation | Status |
|--------|----------|------|------------ |--------|
| Path traversal in `--branch` (`../../etc`) | **T**ampering | High | `strstr(pszBranch, "..")` check at `main.c:185` | Implemented |
| Path traversal in `--output-dir` | **T**ampering | High | `strstr(pszOutputDir, "..")` check at `main.c:189` | Implemented |
| Excessively long argument strings | **D**enial of Service | Low | `getopt_long()` with bounded `snprintf()` copies | Implemented |

## 5. MITRE ATT&CK Mapping

| Technique | ID | Relevance | Scanner Control |
|-----------|-----|-----------|-----------------|
| Command and Scripting Interpreter | T1059 | Attacker injects shell commands via crafted filenames or paths | `fork()/execlp()` eliminates shell interpretation |
| Masquerading | T1036 | Malicious tarball with misleading names or symlinks | `_is_safe_path_component()` validates all path components |
| Obfuscated Files or Information | T1027 | Hidden dependency in deeply nested go.mod or obfuscated spec macros | Full `go.mod` parsing, macro expansion, evidence trail |
| Supply Chain Compromise | T1195 | Poisoned upstream source or modified mapping CSV | Version-controlled data files, read-only operation on inputs |
| Exploit Public-Facing Application | T1190 | CI pipeline runs scanner on untrusted inputs | Input validation on all 6 attack surfaces |
| Input Capture | T1056 | Exfiltration via crafted output paths | Path traversal checks prevent writing outside output directory |

## 6. Risk Summary

| Risk Level | Count | Key Risks |
|------------|-------|-----------|
| Critical | 1 | Tarball path traversal (AS-3) — mitigated |
| High | 4 | Command injection (AS-2, AS-3), CLI traversal (AS-6) — all mitigated |
| Medium | 6 | Buffer overflows, DoS, PRN tampering — all mitigated |
| Low | 5 | Malformed input, CSV tampering — all mitigated |

**Residual Risk**: Low. All identified threats have implemented mitigations. The scanner operates in read-only mode on input files and writes only to the designated output directory.
